LTRATO-1001 | Service Insertion as Code: From Concept to Commit
Key Concepts
Provider
A plugin that knows how to
talk to a specific platform.
This lab uses:
kreuzwerker/docker
CiscoDevNet/iosxe
terraform {
  required_providers {
   iosxe = {
    source = "CiscoDevNet/iosxe"
   }
  }
}
Resource
A single piece of infra
managed by Terraform.
Examples:
docker_container
iosxe_interface_loopback
resource "iosxe_interface_loopback"
    "lo0" {
  name       = 0
  description = "Managed by TF"
}
State File
Terraform's memory —
records what it deployed.
Stored in:
terraform.tfstate
Never edit by hand
# After apply:
$ cat terraform.tfstate
"hostname": "csr-terraform"
"address": "10.99.99.1"
Module
A reusable group of
resources packaged together.
This lab uses:
docker-infra
iosxe-config
module "iosxe_config" {
  source   = "./modules/iosxe-config"
  hostname = var.hostname
}
