No agents installed on devices — Ansible connects over SSH Managed Devices (10 nodes)
Agentless
No software needs to be installed on managed devices. Ansible connects over SSH — the same way you would manually.
Declarative
You describe what the state should be, not the exact steps to get there. Ansible figures out what commands to send.
Idempotent
Running the same playbook twice produces the same result. If the config is already correct, Ansible makes no changes.
Multi-Vendor
One tool handles NX-OS, IOS-XE, IOS-XR, Linux, and more. Platform-specific collections handle the differences.
In this lab, the dCloud server is the control node. You write YAML playbooks, run ansible-playbook, and Ansible handles connecting to every device, sending the right commands, and verifying results — no manual SSH sessions required.
